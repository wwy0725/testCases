# 一、筛选项

1. 枚举获取接口： [14-营销案例库销售方案查询条件枚举](/read/book/70/174254) saleCaseParamEnumList
2. 沟通方式：
    1. 单选，管理端配置枚举。筛选项：不限，有录音，有企微对话，两者都有，空，
    2. 对应查询字段：itcrm\_marketing\_overall\_sales\_case\_info。call\_way
3. 行业门类：多选，管理端配置枚举，对应客户中心的行业大类。对应查询字段：itcrm\_marketing\_overall\_sales\_case\_info。industry\_code
4. 业务分类：
    1. 单选，管理配置枚举，单选；
    2. 筛选项：不限，数智解决方案，（二级枚举：不限、纳新、升版、扩户、平续，单选（可配））税优合规，数智合规，其他
    3. 对应查询字段：itcrm\_marketing\_overall\_sales\_case\_info.calendar\_code
5. 成交时间：支持选择近一年的时间

# 二、列表

1. 查询接口：03-赢销案例列表查询/marketingCase/queryMarketingCaseList
2. 查询范围：返回： 【案例归属大区=当前业务大区 或者 案例的可见范围包含(当前业务大区  或者 当前经营中心）】且 类型是整体销售案例 且 未下架的案例
    1. SELECT  * FROM itcrmweb.itcrm_marketing_case_info WHERE is_delete =0
AND (big_region_code  ='004012022' OR view_range  LIKE '%004012022%' OR  view_range = '004012'   OR  view_range LIKE '004012,%' OR  view_range LIKE '%,004012' OR  view_range LIKE '%,004012,%')
AND sample_en_type ='overallSalesCase'
AND app_code ='itcrmweb'
ORDER BY id DE
3. 排序：
    1. 优先排置顶的案例，按照修改时间倒序
    2. 未置顶的，有录音和企微聊天>至少有录音或企微其中一类的＞沟通过程为空即没有录音也没有企微对话的，想通类型 按照生成时间、id 倒序
4. 案例标题展示：
    1. 业务分类：业务分类无子级，显示业务分类的名称；业务分类有子级，显示为【父级名称-子级名称】
    2. 推荐人：生成案例时取itcrmweb.itcrm\_marketing\_case\_origin\_info.creator\_id，案例生成后，写入 itcrm\_marketing\_case\_info.creator\_id
    3. 日期：生成案例时取itcrmweb.itcrm\_marketing\_case\_origin\_info.generation\_time，案例生成后，写入 itcrm\_marketing\_case\_info.create\_time
5. 显示推荐理由
    1. itcrm\_marketing\_overall\_sales\_case\_detail\_info，value\_type='recommendReason'， value的值
6. 收藏、点赞、置顶、下架，和外呼录音一致 
    1. 参考 [案例列表操作](/read/book/167/200201)

# 三、编辑案例

1. 案例库管理员【具有权限：案例库管理（marketingCaseManage）】且 案例所属大区和登录人业务大区一致，可以编辑案例，和可以编辑方案总结
2. 编辑整体销售案例报告里的四个总结字段，保存编辑次数，[13-编辑营销案例库销售方案](/read/book/70/174255) /marketingCase/updateMarketingSaleCase，存储到itcrm\_marketing\_overall\_sales\_case\_info  表 update\_num 字段，每编辑一次，编辑次数+1




# 四、上传录音

1. 案例库管理员【具有权限：案例库管理（marketingCaseManage）】且 案例所属大区和登录人业务大区一致，可以上传录音
2. 击【上传录音】打开上传录音组件，带入传参：客户经理trueID，联系人personID、名称，案例关联的上门录音ID
3. 如果带入的上门录音ID和录音组件推荐的录音ID相同，自动勾选且置灰
4. 录音组件，上传、删除、推荐录音等功能，有外呼提供：[3.3 关联录音组件](/read/book/70/199569)
5. 上传录音完成 且 销售案例已经人工修改过，显示是否重新生成的确认弹框
    1. 03-赢销案例列表查询/marketingCase/queryMarketingCaseList，updateFalg：itcrm\_marketing\_overall\_sales\_case\_info的update\_num字段不为空且大于0，则为 Y，否则为 N
    2. 上传录音完成  + 销售案例已经人工编辑过（updateFalg='Y'）：显示是否重新生成的确认弹框
    3. 上传录音完成  + 销售案例无人工编辑过(updateFalg='N')，不显示弹框
6. 上传录音之后，未重新生成案例，录音和案例关联，刷新案例详情
    1. [17-营销案例库销售方案关联录音](/read/book/70/174251)/marketingCase/overallSalesCaseRelationCall
    2. 获取itcrm\_marketing\_overall\_sales\_case\_detail\_info 表中 value\_type = visitId 对应数据，删除原数据，原数据+入参外呼 id 保存为新数据
    3. 获取itcrm\_marketing\_case\_call\_record\_relation保存 businessId，案例库 id，呼叫记录 id
    4. 刷新案例详情，显示最新关联的录音，可播放录音
7. 上传录音 且 案例库已经编辑过，重新生成录音
    1. 前端调用[15-营销案例库销售方案重新生成](/read/book/70/174253) /marketingCase/resetOverallSalesCase，入参："marketingCaseId" : "案例库id", 必填， "callRecordIdList" : "外呼id列表"，必填
    2. 查询案例库 id 对应客户 id，调用AI服务的dubbo接口 [01-营销案例库销售方案重新生成](/read/book/70/173204)接口
        1. 将所有数据落地到itcrm\_marketing\_case\_origin\_info表，做下记录；需要将元数据复制一分后，落地call\_record\_id字段
            1. **更新 call\_record\_id** 
            2. **status\_code 置为0**
            3. **retry\_count 置为空**
            4. **batch\_id    置为空**
            5. **remark      置为空**
            6. **marketing\_case\_id 置为空**
            7. **type\_code 置为 sale\_case\_operator**
        2. 异步唤起OverallSalesCaseTask定时任务相关逻辑 
        3. 该场景下还需要去查询‘已上传补充录音’仍然需要调用[查询指定客户经理和联系人的录音](/read/book/70/170430)接口，查询客户经理与联系人的录音信息，两者去重后得到需要处理的录音信息

# 五、案例详情

1. 【编辑】按钮
    1. 案例库管理员【具有权限：案例库管理（marketingCaseManage）】且 案例所属大区和登录人业务大区一致，显示【编辑】按钮
    2. 只有【沟通记录总结，销售方案分析，客户痛点和需求+解决方案，客户疑问/异议处理】四个字段可以编辑
2. 【下载文件】按钮
    1. 无权限控制，点击可下载文件，打开的是已经生成的PDF文档：itcrmweb.itcrm\_marketing\_sale\_case\_info。file\_url
    2. 因为PDF获取的是快照信息，案例详情的几个字段是获取实时信息，所以数据有可能不一致
3. 显示字段
    1. 所属大区：客户所在大区，根据客户 id 调用市场管理接口实时获取
    2. 客户名称：实时获取
    3. 所属多企业：如客户有关联到多企业则显示，实时获取
        1. 根据客户 id 调用客户中心05-查询指定客户在经营体体内的关系详情queryBusinessEntityCustomerRelDetail ，获取单位客户对应多企业
        2. 无关联多企业，字段内容显示为空
    4. 多企业关联户数：实时获取
        1.  客户所属多企业下关联企业数
        2. 无关联多企业，字段内容显示为空
    5. 行业： 客户所属行业大类-中类-小类，取自itcrm\_marketing\_overall\_sales\_case\_info.content
    6. 企业分层： 客户所属企业分层，取自标签（code：company\_layer）加工结果，实时获取
    7. 对接人：客户联系人， 有多个客户联系人用、隔开
    8. 成交金额：itcrm\_marketing\_case\_origin\_info。deal\_amount
    9. 成交产品包：itcrm\_marketing\_case\_origin\_info。product\_info
    10. 成交时间：itcrm\_marketing\_case\_origin\_info。order\_deal\_time
    11. 跟进方式、收起沟通过程
        1. 获取报告生成时往前推6个月客户经理与对接人之间的外呼记录、企微聊天记录以及已上传补充录音（分类为”上门或其他“）
        2. 外呼录音则显示“电话”；
        3.  有企微聊天记录则显示“企业微信”
        4. 有上门及其他录音则显示“上门”
        5. 有多项，叠加显示：电话、微信、上门
        6. 都没有则显示：无
        7. 有录音或者有企微对话，显示【展开沟通过程】按钮，否则不展示
    12. 沟通过程
        1. 默认收起，点击【展开沟通过程】展开，点击【收起沟通过程】收起 
        2. 录音 
            1. 显示6个月内客户经理与对接人之间的外呼录音
            2. 显示所有的上门录音
            3. 外呼录音和上门录音排序：按时间轴顺序显示，精确到年月日时分秒
            4. 显示总共录音个数
            5. 名称拼接：录音【顺序值】-录音类型-时间，如录音2-上门或其他-2025-08-01 18:00
            6. 实时获取录音文本和录音小结
                1. 外呼录音文本：getCallRecordList
                2. 外呼录音小结：getCallRecordDetailList
                3. 上门录音文本和小结：getFilmRecordAndTextList
        3. 企微会话
            1. 对话文本：fetchChatDataDetail
            2. 对话小结：itcrm\_marketing\_overall\_sales\_case\_detail\_info，value\_type='messageSummary'，value字段的值
    13. 沟通记录总结，销售方案分析，客户痛点和需求+解决方案，客户疑问/异议处理
        1. 有值：显示内容
        2. 无值：显示【无】
        3. 可编辑，保存编辑次数，[13-编辑营销案例库销售方案](/read/book/70/174255) /marketingCase/updateMarketingSaleCase，存储到itcrm\_marketing\_overall\_sales\_case\_info  表 update\_num 字段，每编辑一次，编辑次数+1
