# 2、分公司仓库管理

用户配置分公司在各个场景下的 经营动作，具体事项，跟进结果，关联能力的枚举值配置

1、分公司没有自己的配置：显示默认分公司的配置枚举值，**不可编辑；**

2、分公司有自己的配置：显示配置的枚举值内容，四个枚举值项**都可编辑；**

3、只有有自己配置的分公司才能被复制，复制的可选项是有自己配置的分公司列表；

4、判断分公司是否有自己的配置：根据表 om\_enum\_value\_belong 和 om\_service\_form\_config 两个表中是否有分公司的配置数据决定

5、复制分公司的配置，涉及以下数据的复制

- 分公司枚举仓库的配置，包括跟进记录相关的6个枚举：跟进记录场景、关联能力、经营动作、具体事项、跟进结果、跟进方式，对应数据表-om\_enum\_value\_belong，原先自己分公司的配置全部删除

# 3、跟进记录配置

用于配置分公司/部门某个场景下的所有字段的是否显示/是否可以修改/是否跳转

进入到页面，显示为空。只有查询之后才显示具体的场景配置数据；

该公司/部门未配置时，表格只显示表头，内容为空；

查询配置的逻辑：

- 分公司+部门+应用+场景：唯一确定一套配置
- 分公司+应用+场景：唯一确定一套配置
- 分公司+场景：默认选择 分公司+itcrmweb（应用）+ 场景的配置，若当前分公司没有配置时，查询默认分公司的配置
- 小二当家应用的配置是其他应用（传入的appCode）的兜底，即当其他应用（包括中介CRM）查不到配置时，就返回小二当家对应场景的配置

  兜底逻辑简要概述：**根据调用方传参进行查询，查询不到就一级一级往上查，查询到分公司维度，如果还没有结果就替换应用为itcrmweb，重复上述流程，如果还没有，就返回默认分公司对应的配置**

**select** substr(osfc.form\_condition,1,6) **as**'分公司',

substr(osfc.form\_condition,10) **as**'跟进记录场景',

## 3.2 操作（修改，复制，设置联动关系）

3.2.1 修改：

- 修改配置后保存，删除原表中的配置项，新增该场景该分公司/部门的新的配置项，数据保持到[运营支撑管理数据库](http://docs.dc.servyou-it.com/pages/viewpage.action?pageId=134025854)表operationmanageweb. om\_service\_form\_config。

3.2.2 复制：

- 可以复制已经配置过的场景到当前的场景；支持复制分公司下的某应用的某场景或分公司下部门的某应用某场景。
- 复制场景，会把源场景的联动字段，联动关系设置一起复制过来
复制分公司的配置，涉及以下数据的复制
    - 分公司在跟进记录配置下涉及公司级别的配置，不包括部门级别的场景配置，对应数据表-om\_service\_form\_config，原先自己分公司的配置全部删除
    - 分公司在跟进记录配置下设置关联关系的上下级配置，比如“经营动作-具体事项”，对应数据表-om\_service\_child\_enum\_relation， 原先自己分公司的配置全部删除
    - 分公司在跟进记录配置下设置关联关系的具体枚举值配置，比如“经营动作（回电）-具体事项（接通）”，对应数据表-om\_service\_child\_enum\_config， 原先自己分公司的配置全部删除

3.2.3 设置联动关系：

- 设置联动关系字段，比如设置联动的字段为：经营动作，具体事项。设置后的数据保存在[运营支撑管理数据库表operationmanageweb.om\_service\_child\_enum\_relation](http://docs.dc.servyou-it.com/pages/viewpage.action?pageId=134025854)里。
- 可以添加/删除联动关系字段，两组之间不去重。

3.2.4 联动关系具体设置：

- 每个字段的枚举值可以多选，首级枚举值不能重复。已选中的枚举值不会再显示在添加的下拉列表里。
- 非首级的字段，在同一个上一级枚举值下不可重复。
- 在上一级没有选中枚举值时，下一级的枚举值添加按钮置灰不可点击
- 保存时调用接口operationmanageweb/followRecord/saveFollowRecordAllRelation，数据保存在[运营支撑管理数据库表operationmanageweb.om\_service\_child\_enum\_config](http://docs.dc.servyou-it.com/pages/viewpage.action?pageId=134025854)里。
- 可添加/删除/移动枚举值
- 选中上一级枚举值时，联动出该枚举值关联的下一级的所有枚举值，第一个枚举值高亮显示。
- 删除枚举值，会删除该枚举值下的所有关联的下级枚举值

# 4、前端组件的联动关系展示逻辑

4.1 没有包含在其他联动关系里的字段才能显示在首级，如：

| - | - | - |
|---|---|---|
| 配置联动字段 | 添加跟进记录时加载的首级字段 | - |
| A，B，C  B，D | 首级只会显示A字段 | - |
| A，B  C，D | 首级会显示A字段和C字段 | - |
| - | - | - |

4.2 如果字段只有一个枚举值，默认选中，比如【经营动作】只配置了一个枚举值“线索跟进”，添加跟进记录时这个枚举值自动被选中，不需要手动选中

4.3 如果字段有多个枚举值，不会默认选中。

4.4 重新选择枚举值后，之前枚举值关联的字段和枚举值被清空

4.5 上级枚举值选中后，联动出来下一级的字段和枚举值，只显示一条链路上的字段和枚举值

如，配置了联动字段A，B，C，联动枚举值如下

| - | - | - | - |
|---|---|---|---|
| A | B | C | D |
| A1 | B1 | C1、C2 | - |
| A2 | B2 | - | - |

添加跟进记录时，显示字段A，下拉枚举值为A1，A2

选中A1，显示字段B，枚举值为B1，B2

选中B1，显示字段C，显示枚举值C1，C2

选中A2-B2，不会显示字段C和其枚举值




4.6 设置了首级相同的多组联动字段，首级的枚举值合并去重后显示，

如：配置2组联动字段

A B C

A B D

具体枚举联动为

| - | - | - | - | - |
|---|---|---|---|---|
| 联动字段 | A | B | C | D |
| A B C | A1 | B1 | C1、C2 | - |
| A2 | B2 | - | - | - |
| A B D | A2 | B1 | - | D1 |

添加跟进记录时，展示A字段，枚举值显示A1，A2

- 当选择A1时，联动出B字段，枚举值为B1和B2
    - 选择B1时，联动出C字段，枚举值为C1和C2
    - 选择B2时，不联动出任何字段
- 当选择A2时，联动出B字段，枚举值为B1
    - 选择B1时，联动出D字段，枚举值为D1

4.7 其他

- 关联能力做了特殊处理：当关联能力的枚举的枚举code是关联签到的code时，可以联动出“关联签到”组件
