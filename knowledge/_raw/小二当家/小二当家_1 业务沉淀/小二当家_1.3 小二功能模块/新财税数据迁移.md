设计文档：[【20220414】融合-历史数据迁移](/read/book/70/174018)

itcrm\_agency\_data\_sync\_record 表记录迁移前后的业务id关系

执行迁移接口时，需先判断当前登陆者有无 执行数据迁移 的功能权限（权限系统，小二的应用下关联融合数据同步的权限，权限code = agency\_dataSync）

### 商机

迁移规则详见设计文档；

| - | - | - | - |
|---|---|---|---|
| - | 源表 | 目标表 | 备注 |
| 商机主表 | yypt5\_sales.chance\_info | itcrmweb.itcrm\_chance\_info | - |
| 商机动态信息 | yypt5\_sales.chance\_items | itcrmweb.itcrm\_chance\_items | - |
| 最近一条商机 | - | itcrmweb.itcrm\_chance\_last\_info | 自动处理客户的最近一条商机 |




以下几种情况数据不迁移：

1、当前客户在小二存在未完结的商机，则中介未完结的商机不迁移，已完结的商机迁移；

2、当前客户在中介的商机状态无效，不迁移；




注意事项：

1. 原商机动态（yypt5\_sales.chance\_items.ITEM\_CONTENT）
    1. 有跟进内容的迁移成跟进记录并关联商机动态，类型=跟进；
    2. 无跟进内容的直接迁移成商机动态，类型=编辑；
2. 原商机无负责人定义，迁移过程中通过实时查询市场管理接口，补充商机负责人为责任客户经理
3. 商机和线索的关联关系通过线下脚本执行；
4. 历史已迁移过的数据，在新财税中修改过的商机，不会再迁移，如果需要迁移，需要先修改最后修改时间，再重新补迁数据（老的已完结的数据，在新的修复成已完结）
