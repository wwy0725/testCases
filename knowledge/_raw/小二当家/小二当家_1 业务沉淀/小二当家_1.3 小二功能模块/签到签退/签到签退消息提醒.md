# 二、权限配置

有权限的员工可接收消息

| - | - |
|---|---|
| 功能权限 | 作用范围 |
| 下属签到签退提醒 - subordinateSignInSignOutNotice | 下属签到/签退时，给有权限的主管推送企微消息 |
| 非客户经理签到签退提醒 - nonCustomerManagerSignInSignOutNotice | 新增签到/签退时，签到人/签退人非客户的责任客户经理，给有权限的责任客户经理推送企微消息 |




# 三、消息推送逻辑及内容

## 3.1 下属签到签退提醒

### 3.1.1 消息推送逻辑

触发时机：保存签到/签退

1. 获取有【下属签到签退提醒】权限的员工
    1. 根据权限查询员工未返回的结束，不推送消息
2. 循环判断当前签到人/签退人是否是以上员工的下属，获取到推送对象列表
    1. 获取有权限的员工的数据权限 
        1. 无数据权限的，结束
    2. 根据数据权限的部门获取下级部门得到部门列表
    3. 判断签到人/签退人的组织架构code是否在部门列表内，在则表示是下属
    4. 循环判断结束都不是以上员工下属的，不推送消息
3. 按推送对象列表，循环推送消息
4. 下属签到签退提醒消息记录查询select
	*
from
	itcrmweb.itcrm_sale_message_notification
where
	EVENT_TYPE in (
'subordinate_sign_in_notice',  -- 下属签到提醒
'subordinate_sign_out_notice'  -- 下属签退提醒
)
order by
	id desc;

如果签到人/签退人有数据权限且包含本人所在的部门，也会收到下属签到签退提醒

#### 3.1.2.2 消息内容

- 客户名称：展示客户全称，超过换行；
- 签到人：展示签到or签退人的名称
- 签到时间：签到or签退的时间，格式：yyyyy-mm-dd hh:mm:ss
- 签到地址：展示签到or签退的具体位置的文本信息，超过换行
- 展示是否范围内签到or签退
    - 范围内，则展示“范围内签到”or“范围内签退”
    - 范围外，则展示“范围外签到”or“范围外签退”，并展示具体原因，拼接在后面，格式“（原因+具体内容）”，超过换行
- 签退提醒时展示停留时长，格式“停留时间：时间间隔”
    - 超过1天，显示“大于1天”
    - 超过1小时显示N小时
    - 超过1分钟显示N分钟
    - 不到一分钟显示“小于1分钟”
- 消息链接
    - 签到提醒：打开签到详情
    - 签退提醒：打开签退详情，定位到签退的tab
    - 消息链接需要携带appCode：根据客户经理的经营中心映射appCode
        - 中小-004012 或 新疆-004013 或空：itcrmweb
        - 财代-004011：agencycrmweb
        - 政务-004010002：gitcrm
        - 人资大客-004010001：ptcrm
- 业务目的：只有财代业务线提醒的消息中会包含该字段
- 下一步计划：只有财代业务线提醒的消息中会包含该字段







## 3.2 非客户经理签到签退提醒

### 3.2.1 消息推送逻辑

触发时机：保存签到/签退

1. 获取签到/签退的客户
2. 获取客户的责任客户经理
    1. 市场管理接口：[01-根据客户id列表批量查询划分结果](/read/book/70/172794)
    2. 客户无责任客户经理的结束，不推送消息
3. 判断签到人/签退人是否=客户的责任客户经理
    1. 签到人/签退人 = 责任客户经理的结束，不推送消息
4. 判断客户的责任客户经理是否有功能权限：非客户经理签到签退提醒 - nonCustomerManagerSignInSignOutNotice
    1. 无权限的结束，不推送消息
5. 满足以上条件，给客户经理推送企微消息
6. 非客户经理签到签退提醒消息记录查询select
	*
from
	itcrmweb.itcrm_sale_message_notification
where
	EVENT_TYPE in (
'non_customer_manager_sign_in_notice',  -- 非客户经理签到提醒
'non_customer_manager_sign_out_notice'  -- 非客户经理签退提醒
)
order by
	id desc;

## 3.3 未签退提醒

### 3.3.1 消息推送逻辑

触发时机：定时任务执行扫描指定时间段内未签退的签到记录，判断是否需要签退已经开启消息通知，符合条件的推送未签退提醒

#### 3.3.1.1 配置项

| - | - | - |
|---|---|---|
| 配置方式 | 配置 | 说明 |
| 运营支撑管理端 | 签到管理/未签退提醒(notSignOutRemind) | 按appCode+大区配置开关 |
| iris | not.sign.out.remind.minutes=120 | 签到未签退提醒时间间隔，N分钟内未签退（单位：分钟） |
| iris | not.sign.out.remind.range.minutes=30 | 签到-未签退提醒任务查询时间范围，往前扫多少分钟（单位：分钟） |
| iris | not.sign.out.remind.redis.expire.minutes=1440 | 签到未签退提醒已发送记录redis超时时间（单位：分钟），已发送的签到id设置到缓存里，超时自动销毁 |




#### 3.3.1.2 定时任务执行

LTS任务id：NotSignOutRemindTask，配置每一分钟执行一次

测试环境小二任务执行节点：test%job\_group\_itcrm

release环境小二任务执行节点：job\_group\_itcrm    

1. 获取指定时间范围内的未签退（itcrm\_sign\_in\_info.IS\_SIGN\_OUT=N）的签到记录
    1. 限制签到起止时间：
        1. 起：定时任务当前执行时间减去签到未签退提醒时间间隔再减去往前扫多少分钟的配置（为了防止漏数据）
        2. 止：定时任务当前执行时间减去签到未签退提醒时间间隔，取配置not.sign.out.remind.minutes，配置120即表示取最后签到时间是2小时前的签到记录
        3. 举例说明：2026-04-10 11:00:00执行定时任务，本次查签到记录的时间范围为 11点往前推2小时再往前推半小时 \~ 11点往前2小时的签到记录，即2026-04-10 08:30:00 \~2026-04-10 09:00:00之间未签退的签到
        4. not.sign.out.remind.range.minutes配置0的话表示不设置时间富余量，只查120分钟前这一分钟之内的签到
2. 获取到指定时间范围内的签到id，判断是否已经在已推送过消息的缓存内，过滤在缓存的签到id，避免重复发送
3. 过滤后的签到记录循环判断是否符合消息推送条件
    1. 获取配置根据签到人的信息，取签到人的经营中心映射到对应的appCode，大区取签到人的大区
    2. 根据appCode+大区查询未签退提醒的配置是否开启，未开启的结束
    3. 已开启的判断当前大区是否配置了无需签退：select \* from itcrm\_system\_config t WHERE t.key = 'signType'，无需签退的不推送消息
    4. 大区开启了未签退提醒且需要签退的给签到人推送消息，把对应的签到id设置到缓存里
        1. itcrm\_sale\_message\_notification表保存消息记录，EVENT\_TYPE = 'not\_sign\_out\_remind\_notice'

测试接口：

1. /test/getCache，prefix=notSignOutRemind:sent：查询缓存中已发送的签到id
2. /test/delCache，key=notSignOutRemind:sent:912859：根据签到id删除缓存记录
3. /test/notSignOutRemindTask：任务测试接口
    1. triggerTime：设置任务触发时间，格式：2026-04-09 20:24:00
    2. remindMinutes：对应配置 not.sign.out.remind.minutes
    3. rangeMinutes：对应配置 not.sign.out.remind.range.minutes
