## 头部模块

### 字段展示

1. **客户名称：**商机主体的名称（企业、个代、中介、多企业），个人商机不展示该字段
2. 有产品包的情况下，头部显示产品包信息：产品包名称（预计销售金额），产品包为空时不显示。
3. 调用sales/chance/detail接口获取详情信息，调用sales/chance/stage/list获取销售阶段信息；
4. 联系人：展示商机联系人+手机号/固话
    1. 如果商机包含其他联系人，在主联系人后面展示“更多”按钮，点击打开弹框，展示其他联系人的名字，支持带appCode跳转一人式 （20240321迭代：目前仅大客商机有其他联系人）
    2. 联系人后面展示外呼组件：
        1. 联系人存在手机号时，传手机号给外呼组件
        2. 联系人不存在手机号，但是存在固话时，传固话给外呼组件
        3. 联系人不存在手机号和固话时，不展示外呼组件
5. mob经营体客户名称不可点击
6. 展示客户的类型标签（个代/中介/企业/经营体）
7. 商机详情页展示“业务分类”字段，有值展示，无值隐藏
    1. 人资大客的商机不展示“客户成功”
8. 协同人员：有值展示，无值隐藏，取 itcrm\_chance\_collaboration\_info 表信息
9. 原到期产品信息：格式：”原产品到期信息：原到期产品（yyyy-mm-dd到期）”
    1. 取 itcrm\_chance\_original\_expired\_package\_info 表中数据
    2. 可能存在多个，逗号隔开
10. 重点关注标记：具体参考 [重点关注标记（需求）](/read/book/70/204764) 
11. 休眠状态：如果商机休眠状态=已休眠，展示“已休眠”的标记，如果未休眠，不展示任何标记
12. **商机阶段**：取编辑场景配置的商机阶段枚举（因为商机阶段可能与类型联动配置，所以需要传入formCondition作为条件）
    1. 接口：/sales/chance/getSaleChanceModuleFormDetailConfig
    2. 入参：
        1. 大区：前端不传（服务端取登录人所属的大区）
        2. 场景：写死“编辑场景update”
        3. 商机对象、业务线、商机类型、商机阶段：取商机详情数据

### 按钮

1. **编辑：**商机负责人，或有编辑下属权限（editUnderChance）的负责人上级（具体参考[主管编辑下属商机](/read/book/70/204764)）展示该按钮
2. **跟进：**商机负责人展示
3. **上级建议：**按钮展示条件详见 [上级建议](/read/book/70/204764) 
4. **商机指派：**按钮展示条件详见 [商机指派](/read/book/70/204764)
5. **下单**：upms授权权限“财代商机快捷下单”（code=agencySaleChanceOrder）时，详情也展示下单按钮，暂不限制客户类型及应用（线上应用在财税），对接商务下单组件，与机构一户式一致，但商机详情页面取不到税友号，因此无需传税友号
6. **签到/签退（仅mob移动端商机详情页）**：符合条件时展示签到/签退按钮，支持对商机主体对象进行签到/签退，需要满足的条件如下
    1. 商机主体类型不是个人
    2. 06-快捷判定签到/签退接口返回签到/签退 ItcrmSignFacade.customerSignInOrOut（传参：登录人所属大区+appCode+人+客户）
7. **转多企业商机**：参考沉淀 [中小企业商机转多企业商机](/read/book/70/204764) 
    1. detail详情接口  isShowParseBusinessChance === Y 是展示
8. **报告解读**：
    1. detail 接口 isShowReport 为Y时展示 
    2. 按钮展示条件：
        1. 商机业务分类=税优合规
        2. 商机对象是企业
        3. 商机有协同人员或已创建解读报告
    3. 按钮点击时，前端调用接口[根据企业ID查询是否有在途报告解读](/read/book/70/174722)（/customer/hasMovingReport）
        1. 若返回 true，表示有在途数据，对接商务报告解读列表组件，展示客户关联的报告解读数据
        2. 若返回 false，表示无在途数据（从未创建过或都已完结），对接商务报告解读创建组件
            1. 若上级只有1个协同人员，直接带入解读报告
            2. 若商机有多个协同人员，需要弹出选择框：可选人员为商机已添加的协同人员
        3. 前端对接商务组件传入参数：
            1. 商机id+商机对象（企业）+联系人+商机已选的协同人员
    4. 商务组件：
        1. 创建报告解读时商务会校验客户，若客户不满足条件会提示需要先通过安全认证（在其他平台，比较麻烦，建议直接用现成数据）
        2. 报告解读创建后可以在[专家工作台](http://reception-manage-pc.base-service-test.sit.91lyd.com/reception-manage-pc/#/expertsServicePlatform/receptionPlatform?tabActiveKey=service) 进行完成操作
            1. 若打开空白，去控制台加一下对应应用的权限即可

## 商机动态

| - | - |
|---|---|
| 字段 | 取值逻辑 |
| 操作人 | - |
| 操作类型 | 目前展示的操作类型有：  新增、修改、跟进、关联、指派、转多企业商机、批量指派（batchAppoint） |
| 操作时间 | 精确到分钟 |
| 商机阶段 | 显示“名称（百分比）” |
| 联系人 | 有跟进记录，展示跟进记录联系人；  没有则展示商机编辑时的商机联系人  （超过长度展示“...”，可跳转一人式） |
| 赢单/输单原因 | 展示关联的合同/订单+赢单/输单原因；  订单支持跳转详情页  标准合同抽屉打开，非标合同跳转页面 |
| 预购产品包 | 产品包名称（预计销售金额） |
| 休眠状态 | PC端：已休眠/未休眠  移动端：状态=已休眠时展示，状态=未休眠时不展示对应字段 |
