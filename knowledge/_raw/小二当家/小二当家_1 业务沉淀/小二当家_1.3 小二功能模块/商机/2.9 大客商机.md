# 商机流程

参考文档：[大客商机产品需求](http://proto.dc.servyou-it.com/UIfiles/02-%E8%90%A5%E9%94%80%E6%9C%8D%E5%8A%A1%E7%BB%84/%E8%BF%90%E8%90%A5%E6%94%AF%E6%92%91%E4%B8%AD%E5%BF%83%E9%9C%80%E6%B1%82%E6%B1%87%E6%80%BB/%E5%B0%8F%E4%BA%8C%E4%B8%AD%E4%BB%8B%E4%BA%92%E9%80%9A%E4%B8%93%E9%A1%B9/start.html#id=d0v34n&p=%E4%BA%BA%E8%B5%84%E5%A4%A7%E5%AE%A2%E5%95%86%E6%9C%BA%E5%95%86%E6%9C%BA%E6%95%B4%E4%BD%93%E8%AF%B4%E6%98%8E&g=1)

大客商机的特殊点：

- 商机配置走运营支撑管理端新功能，配置操作可参考文档：sh
- 人资大客创建、编辑、跟进、关联场景的展示字段及校验逻辑对接运营支撑管理平台的新配置后台
    - 操作场景：创建、编辑、跟进/关联三种sho
    - 支持按照appcode+大区+商机对象+操作场景+商机类型+商机阶段需要展示的字段、是否必填、枚举值、或选择某个枚举值再展示
    - 支持按照阶段配置“推进校验项”，供商机判断是否可以进入下一阶段
    - 支持配置字段展示顺序，各字段顺序以运营支撑管理端配置顺序为准
- 人资大客所有商机都不允许跳阶段，只能逐级推进
- 人资大客支持根据客户对象+商机类型配置商机阶段
    - 例如平续的商机，可以不设置解决方案阶段；其他类型的商机需要设置所有阶段
    - 本期人资大客商机对象只有中介机构

## 阶段主流程：

- 创建时自动选中第一阶段，无法修改
- 阶段需要逐级推进，无法跳过
- 编辑时，可以选中的阶段为当前阶段的下一阶段或输单，若下一阶段配置有推进条件，需要满足推进条件才能选中，但是任何阶段都可以输单

| - | - | - |
|---|---|---|
| 商机阶段 | 该阶段展示的数据（推进校验） | 对应数据可选中的阶段 |
| 商机阶段0：主管评估（创建时阶段） | 创建时将主管评估结果从空置为待评估 | - |
| 主管评估结果：通过 | 可以推进到阶段1（确认意向），或输单 | - |
| 主管评估结果：不通过 | 无法推进到阶段1（确认意向），可以推进到输单 | - |
| 商机阶段1：确认意向 | 类型是平续 | 可以推进到阶段3（商务谈判），或输单 |
| 类型不是平续，售前评估：待评估 | 无法推进到阶段2（解决方案），可以推进到输单 | - |
| 类型不是平续，售前评估：否 | 可以推进到阶段2（解决方案），或输单 | - |
| 类型不是平续，售前评估：是，且关联了售前项目id | 可以推进到阶段2（解决方案），或输单 | - |
| 商机阶段2：解决方案  （平续类型没有该阶段） | 两种情况满足其一：  1、售前评估：否  2、售前评估：是，且关联的售前项目状态是“已交付” | 可以推进到阶段3（商务谈判），或输单 |
| 售前评估：是，且关联的售前项目状态是“交付中”或者“已取消” | 无法推进到阶段3（商务谈判），可以推进到输单 | - |
| 商机阶段3：商务谈判 | 两种情况满足其一：  1、是否标准报价：否  2、是否标准报价：是 且 关联的报价单已审核通过/无需审核 | 可以推进到阶段4（合同签约），或输单 |
| 是否标准报价：是 且 关联的报价单未审核通过 | 无法推进到阶段4（合同签约），可以推进到输单 | - |
| 商机阶段4：合同签约 | 关联合同的签署状态是待签署或已签署 | 可以推进到阶段5（赢单），或输单 |
| 关联合同的签署状态不是待签署或已签署（包括空） | 无法推进到阶段5（赢单），可以推进到输单 | - |
| 商机终态5：赢单 | 关联合同审核通过且非签署失败 | 可以赢单 |
| 关联合同不满足审核通过且非签署失败 | 无法赢单，可以输单 | - |
| 商机自由终态：输单 | 任意环节都可输单，本期输单原因手动输入 | - |

## 商机推进校验

商机支持在运营支撑管理端设置：推进到当前阶段必须满足的校验项及具体的值

| - | - |
|---|---|
| 主管评估通过 | managerAdopt |
| 售前评估通过 | preSaleModule |
| 解决方案完成 | solutionModule |
| 报价审批通过 | quotationModule |
| 非签署失败且审核通过合同 | contractModule |

- 主管评估
    - 下一阶段的推进校验设置“主管评估”后，当前阶段将主管评估状态从无置为“待主管评估”
    - 主管评估为“通过”，可以推进到下一阶段
    - 主管评估为“不通过”，不可以推进阶段，商机负责人编辑后将评估状态重新置为“待主管评估”
    - 主管评估后，不论是否通过，都会发送消息给商机负责人pi
-  解决方案评估
    - 创建解决方案评估：如果当前操作场景的下一阶段，配置有解决方案通过为推进条件，则创建解决方案商机 ，并发消息提醒有评估权限的人（登录人的**解决方案数据权限**包含当前商机所属部门即为有权限）
    - 售前主管或售前需要评估结果为需要售前介入或不需要售前介入，有评估结果后商机负责人才可以推进到下一个阶段
        - 需要售前介入的商机，需要售前上传售前解决方案及填写售前相关信息，需要在商机看到进程及信息
- 解决方案 
    - 在解决方案阶段，需要评估的售前提供解决方案并完成方可进入下一个阶段。解决方案阶段需要在商机看到相关信息
- 报价审批通过：两种情况满足其一
    - 是否标准报价：否
    - 是否标准报价：是 且 关联的报价单已审核通过/无需审核
- 非签署失败且审核通过合同
    - 非签署失败且审核通过合同，满足条件时才能赢单
（根据合同编号通过财务共享平台查询合同信息，主要是查询合同签署状态）

## 商机字段：

- 项目分析表：枚举值（运营支撑管理端配置配置）
- 业务类型：二级联动枚举（2025-9迭代变更）
    - 保存字段：business\_type、sub\_business\_type
    - 二级联动枚举配置参考 [业务类型：二级联动枚举](/read/book/70/204779)
    - 接口： /common/getModuleFormEnumLinkageTreeConfig
- 产品使用类型：枚举值：自用/非自用
- 客户需求痛点：文本输入，限500字
- 客户预算：数字，金额，保留2位小数
- 项目状态：枚举值（运营支撑管理端配置配置）
- 客户紧迫度：枚举值：紧急、活跃/正在做、着手引入、以后再说
- 卡点类型：枚举值，单选
- 产品类型：枚举值，单选
- 目前使用系统：枚举值（运营支撑管理端配置配置）
- 是否有竞品：枚举值（运营支撑管理端配置配置）
    - 选中“是”时，出现子表单
        - 竞争态势：枚举值（运营支撑管理端配置配置）
        - 竞争对手：文本输入，限1000字
- 客户采购方式：枚举值（运营支撑管理端配置配置：公开招标、单一来源、直接签约、无需签订合同）
    - 当客户采购方式为“公开招标/单一来源”时联动展示子表单： 
        - 项目资料清单：文本框（限500字）+文件上传（最多10个、不限制格式、最多100M，移动端只能上传图片）
        - 客户招标文件：文件上传，支持单个或者多个，最多10个，不限制格式、最多100M，移动端只能上传图片
        - 招投标备注：文本输入，限1000字
        - 附件信息保存在 itcrm\_business\_append\_info 表中
- 解决方案模块：字段如下
    - 是否需要解决方案：初始取解决方案评估状态“待确认”，若解决方案评估状态为已处理，则获取是否需要解决方案字段
    - 解决方案负责人：已处理且需要解决方案，展示售前项目负责人姓名
- 解决方案模块：展示前提：必须关联售前项目，未关联则配置了也不展示且不做校验，若展示字段如下
    - 解决方案进度：枚举值（取售前项目状态：交付中，已交付，已取消）
    - 关联售前项目：对应售前项目的名称，点击可打开售前项目详情 
- 已实现收入**：**从数据组标签服务获取，数据组接口（[21-通过标签code查询机构的标签接口（单个）](https://docs.dc.servyou-it.com/pages/viewpage.action?pageId=132883226)）数据组对应code： **hr\_big\_customer\_income**
    - SELECT \* FROM ADS\_AGENT\_TAG t WHERE t.AGENT\_ID = '23FC022AE7E44237A81E23C07A0A5354';
- 潜在总收入：商机不做保存，对接新客户中心
    - 表单不展示时，不会变更机构的“潜在总收入”，只有用户切实修改数据之后，才会变更该数值 
    - 查询接口：[查询客户详情（根据批量客户ID）dubbo接口](/read/book/70/170659)
    - 更新接口：[修改中介结构/个人代理dubbo接口](/read/book/70/170671)
- 其他联系人+决策链信息
    - 数据保存在 itcrm\_chance\_decision 表中，is\_show 字段表示该联系人是否编辑过决策链信息，用于决定前端展示时展示“暂无数据”还是空字段
    - 其他联系人的可选项：与主联系人一致并去除主联系人
    - 配置是否必填是否可改：必需要编辑的联系人包含联系人+其他联系人，按照顺序展示，只要展示了每个人的信息都填
    - 联系人和其他联系人均支持固话
- 预计回款日期和预计本年回款金额
    - 创建，编辑和关联商机组件，在商务谈判阶段新增预计回款日期和预计本年回款金额，日期精确到年月日
    - 商机基础表和动态表均需要保存
- 是否标准报价：否，支持上传附件，itcrm\_business\_append\_info

## 获取配置接口相关逻辑（前端、服务端）：

- 子表单接口（[/common/getModuleChildFormItemConfig](/read/book/70/175029)）：前端获取子表单配置时，只需要传基本条件+展示条件，比如子表单，条件入参只有基本条件+商机类型、是否KA（项目分析表：需要/不需要）
- 枚举值接口（[/common/getModuleEnumSceneList](/read/book/70/175030)）：获取枚举值只有这一个接口，目前大客商机的枚举值只会配在商机表单下面（formCode: "saleChanceForm"）
- 商机表单（[/common/getModuleFormItemConfig](/read/book/70/175027)）：商机主表单控制展示并受所有条件限制，因此获取主表单配置时需要传入基本条件+所有的自定义条件（对象、类型、阶段、角色），如果某个条件为空，就传空
- 可选阶段接口（[/sales/chance/getAvailableStageCode](/read/book/70/174383)）：大客商机限制阶段只能逐级推进，商机阶段与类型息息相关，因此传入条件为基本条件+决定条件（对象、类型、当前阶段），服务端处理逻辑如下：
    - 根据类型获取运营支撑管理端的枚举和小二阶段配置表（itcrm\_chance\_stage\_settings）中对应的百分比获取当前类型的所有阶段
    - 根据当前传入的阶段，获取下一个阶段+输单阶段，返回当前阶段、下一阶段、输单三个枚举值选项

20240425迭代新增自定义条件 角色（枚举值：客户成功 customerSuccess）：

- 角色这个自定义条件，只有**查询主表单配置**时，会传入（如果有的话），查询子表单配置和枚举（包括商机阶段）时，都不需要传入
    - 通过主表单配置“是否校验”展示与否控制推进校验
- 获取登陆者的角色：
    - 接口：/sales/chance/getChanceRoleCode 返回登陆者有的配置中的功能权限，
    - 配置看iris（支持配置多个，以英文逗号分隔，如果配置多个，以iris顺序为准遍历，返回upms配置的第一个功能权限code）
iris配置：sale.chance.role.code=customerSuccess
- 创建商机时，前端调用 [/sales/chance/getChanceRoleCode](/read/book/70/174374) 接口，获取登陆者的角色数据，查询并展示对应的表单（主表单配置），最后保存到 itcrm\_chance\_info.role\_code 字段
- 编辑商机时，前端根据 /sales/chance/detail 接口返回的数据，获取商机的 role\_code 和原有条件查询对应的表单（主表单配置）
- 总结：创建时，前端查询角色权限并保存到对应商机，编辑时，根据商机保存的role\_code判断（为了保持商机编辑过程中的前后一致性）
- 没有任何角色权限时，查询主表单配置的condition中，role\_code字段传空

# 详情页

1、人资大客商机新增商机详情tab，展示人资大客相关字段

- 如果输入的文字有换行样式，查看详情时需要保留该样式
- 关联报价
    - 展示已关联报价单编号
        - PC端点击新页面打开报价单详情，由商务提供报价单详情
        - 移动端本期因商务无移动端能力，暂不跳转

    - 由商务提供根据报价单编号查询报价单状态能力

    - 展示已关联的合同编号
    - PC端支持点击合同编号新页面打开合同详情
        - 标准合同详情由商务提供
        - 非标合同详情由财务共享平台提供
- 合同审核状态、合同签署状态
    - 由财务共享平台提供根据合同编号查询合同审核状态、合同签署状态的能力

    - 商机状态非完结状态，若已完结：toast提示2s“无法操作！当前商机已完结”
    - 变更范围是机构组内其他机构且不能是当前主体，若不满足：toast提示2s“无法操作！机构组关系不存在”
    - 判断修改后的主体是否存在对应类型的未完结商机，若不满足：toast提示2s“无法操作！变更后机构已存在该类型的商机”

    - chance主表 itcrm\_chance\_info：customerId、customerType、mainId、mainType
    - 关联合同表 ptcrm\_chance\_relate\_contract：都置为无效
    - 商机动态 itcrm\_chance\_items：operationType （modifyMain - 修改主体），MainId（修改后主体id），MainType（修改后主体类型）

    - 业务场景：creationScene = 【modifySaleChanceMain - 变更商机主体】
    - 操作人：operator = 当前登录人
- 变更成功后，发送MQ：topic：itcrmweb-chance-modify-main，消费方如下：
    - 商机关联的解决方案：变更对象
    - 商机关联的报价单：变更对象
