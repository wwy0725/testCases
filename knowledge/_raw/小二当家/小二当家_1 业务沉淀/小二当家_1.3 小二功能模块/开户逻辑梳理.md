# 新增开户

机构一户式开户组件的新增开户按钮权限控制（客户中心运营侧应用下，开户功能权限group\_open）

接口：/page/customer/openAccount/open

调用运营客户中心CustomerFacade#getCustomerDetailByIds查询客户信息，各类客户的entityNo取值如下

中介：机构社会信用代码

单位客户：社会信用代码> 税号

个代：手机号

1. 判断是否为空，客户ID查询不到客户信息则抛出错误“客户不存在”
2. 校验乐代平台开户，手机号必须和个代唯一识别号保持一致（即判断客户信息返回的entityNo和开户的手机号adminMobile），否则抛出错误“负责人手机号必须与唯一识别号相同”
3. 校验客户信息返回的entityType1-法人主体，并且entityNo是XNSH开头，则抛出错误“开户失败，开户前需完善社会信用代码！”
    1. 改为entityNo是XNSH开头，则抛出错误“开户失败，开户前需完善社会信用代码！”
4. 判断客户是否有特定的saas code授权
    1. 根据客户类型customerType获取对应的saas（对应IRIS配置customer.type.saas）
    2. 如果客户类型customerType对应的saas包含DZPT\_JZFW，并且客户信息的bizRegionCode在IRIS配置（account.open.auth.ignore.area.code）内，则不做再做后续校验
    3. 后续校验逻辑：
        1. 测试是否调用dubbo接口[cn.com](http://cn.com/).servyou.business.bgw.interim.facade.EmpowerInfoInterimFacade#listEmpowerInfo即可
5. 开户平台获取，接口/page/customer/openAccount/platform，判断是否为空，客户ID查询不到客户信息则抛出错误“客户不存在”，使用客户信息里的客户类型customerType获取有效的开户平台，对应IRIS配置customer.type.open.platforms




# 停启用

接口：/page/customer/openAccount/status

若传入类型operateType为enable("3", "启用")，则会走到/page/customer/openAccount/open的公用方法，区别在此处使用只有1、4没有2、3

# 维护负责人/修改超管

接口：/page/customer/openAccount/admin

1. 判断是否为空，客户ID查询不到客户信息则抛出错误“客户不存在”
2. 客户类型为3时，校验开户平台是否为01代账，不是则抛出错误“修改负责人手机号，不支持的平台类型”（逻辑原因：仅限代账平台，可以修改个代负责人）

# 客户信息查询

接口：/page/customer/queryById，调用CustomerFacade#getCustomerDetailByIds

## MQ消费

客户中心合并客户发送mq，处理oc\_customer\_open\_info

被合并客户的开户记录转移至主合并客户，若开户平台重复，则仅保留主合并的开户记录，被合并的删除

运营客户中心个代主手机号变更时，消费代理变更MQoperation-customer-agent-info-modify，变更开户负责人手机号，处理逻辑如下

```java
客户类型不为个代的不处理，客户类型前后不一致的不处理，新老手机号有一个为空的不处理，新老手机号一致的不处理
开始处理：
获取个代的开户日志记录
开户记录中开户平台不为"106", "代账个代版"和"102", "助手"的不处理
获取开户记录中adminAccountId对应的原手机号，对接接口：cn.com.servyou.uc.uic.portal.dubbo.sdk.facade.account.v4.AccountFacade#getById
原手机号与个代的新唯一识别号一致的不处理
获取个代新唯一识别号对应的账户信息，对接接口：cn.com.servyou.uc.uic.portal.dubbo.sdk.facade.account.v4.AccountFacade#getAccountByLoginName
账户信息为空：
注册账户信息返回账户ID，对接接口：cn.com.servyou.uc.uic.portal.dubbo.sdk.facade.register.v4.RegisterFacade#simpleRegister
账户信息非空的，直接取账户ID
账户ID为空的不处理
否则修改开户记录中的adminAccountId
```
