### 3.1.2 定时任务执行校验

*说明：预计释放和定时任务释放的客户范围是大区下有责任客户经理且是非会员的财代客户，其中会员类型依赖数据组的会员加工，因此为了保障计算数据的准确性，执行任务时需要确保数据组当天的会员计算已完成，否则取消本次释放计算*

*实现方式：数据组每天会员加工完成后，推送一条MQ消息，小二监听该消息，并写入缓存，缓存key的格式为 qqt\_vip\_config\_finish\_年月日。执行释放任务时先判断Redis中当天是否有指定key的缓存，有则继续执行任务；若未获取到缓存，需要给相关开发和产品推送一条企微消息提醒，推送对象的trueId配置在iris，配置项：not.release.message.receiver*

    2. 设置枚举值时需要手动和跟进记录的枚举code保持一致，例如配置经营动作=上门，需要取跟进记录-经营动作【上门】枚举值的code

    - 当前日期-跟进日期＜配置值，预计释放天数=配置值-（当前日期-跟进日期）

说明：本规则校验客户的有效商机，客户经理需要在一段时间内持续维护商机

1、调用商机提供的方法，获取符合条件的商机最后修改时间

- 限制appCode=agencycrmweb
- 传入客户id、客户类型
- 传入商机负责人=客户的责任客户经理
- 有多条商机的，获取最大的最后修改时间
- 获取商机最后修改时间参考sql
    - 获取客户的最近一次商机变更时间-- 获取客户的最近一次商机变更时间 MODIFY_TIME
select *
from itcrmweb.itcrm_chance_info
where customer_id = '9EC2B0A9BDE84241ABF67EE8BC14A173'
and PRINCIPAL_TRUEID  = 'DF910C299FBE1BDD4AC92093ECC77B2A' -- 客户的责任客户经理
and APP_CODE = 'agencycrmweb'
order by id desc;

2、判断商机最后修改时间和认领时间

- 未获取到商机最后修改时间（不存在商机）或 商机最后修改时间＜认领时间，则取认领时间判断，认领日期查询见[3.2.1 客户的认领/分配时间](#id-财代客户管理策略（新）-3.2.1客户的认领/分配时间)
    - 当前日期-认领日期 ≥配置值，符合自动释放条件
    - 当前日期-认领日期＜配置值，预计释放天数=配置值-（当前日期-认领日期）
- 有获取到商机最后修改时间
    - 当前日期-商机最后修改日期≥配置值，符合自动释放条件
    - 当前日期-商机最后修改日期＜配置值，预计释放天数=配置值-（当前日期-商机最后修改日期）







## 3.5 商机阶段规则校验

说明：本规则校验客户的商机阶段变更，客户经理需要在一段时间内持续推进商机阶段

1、调用商机提供的方法，获取符合条件的商机阶段停留时间，只查询未完结的商机

- 限制appCode=agencycrmweb
- 传入客户id、客户类型
- 传入商机完结状态=未完结
- 传入商机负责人=客户的责任客户经理
- 获取商机阶段变更时间参考sql
    - 获取客户商机阶段变更时间-- 获取客户商机阶段变更时间 STAGE_MODIFY_TIME
select *
from itcrmweb.itcrm_chance_info
where customer_id = '9EC2B0A9BDE84241ABF67EE8BC14A173'
and PRINCIPAL_TRUEID  = 'DF910C299FBE1BDD4AC92093ECC77B2A' -- 客户的责任客户经理
and END_STATUS = 'N'
and APP_CODE = 'agencycrmweb'
order by id desc;	

STAGE_MODIFY_TIME 更新条件：
1、新增商机会记录阶段变更时间
2、维护商机只有前后阶段不一致的时候才会更新阶段变更时间

2、判断商机的返回结果，只取有唯一未完结商机的客户参与规则计算

- 无符合条件的商机
    - 跳过该规则不做校验，且认为不满足释放条件，无预计释放时间
- 有符合条件的商机且数量＞1，即客户有多个未完结商机
    - 跳过该规则不做校验，且认为不满足释放条件，无预计释放时间
- 有符合条件的商机且数量＝1，即客户有唯一的未完结商机
    - 获取商机的阶段变更时间参与计算

3、判断商机的阶段变更时间和认领时间

- 商机的阶段变更时间 ≤ 认领时间，则取认领时间判断，认领日期查询见[3.2.1 客户的认领/分配时间](#id-财代客户管理策略（新）-3.2.1客户的认领/分配时间)
    - 当前日期-认领日期 ≥配置值，符合自动释放条件
    - 当前日期-认领日期＜配置值，预计释放天数=配置值-（当前日期-认领日期）
- 商机的阶段变更时间 ＞认领时间

    - 当前日期-商机阶段变更日期≥配置值，符合自动释放条件
    - 当前日期-商机阶段变更日期＜配置值，预计释放天数=配置值-（当前日期-商机当前阶段变更日期）










## 3.6 有效订单/合同规则校验

说明：本规则校验客户的有效订单/合同，客户经理需要在指定时间内完成下单或签署合同，本规则和其他规则不同，不需要做持续动作，只要有一次规则达成则不再会触发释放

根据客户id调用商务接口查询已签署合同和已收讫订单，两个接口任一有返回符合条件的数据即认为满足校验；返回结果都为空/返回数据不符合条件的则认为不满足校验

有效订单/合同的定义：认领之后客户上产生的已收讫订单或者已签署的合同，不限制下单人和合同签署人
