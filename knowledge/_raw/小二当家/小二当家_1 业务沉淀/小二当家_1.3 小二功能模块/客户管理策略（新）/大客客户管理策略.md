## 一、规则配置

目前仅数据库配置

新客户管理策略按经营中心+大区+客户分层配置规则， 如只设置经营中心级别的规则，大区设置为default，表示经营中心下的大区通用

配置表：

itcrm\_customer\_manage\_rule\_open

设置规则类型和规则开启状态

itcrm\_customer\_manage\_rule\_config

设置各客户分层对应的规则类型的限制值，如客户分层未配置规则，则表示不受客户管理策略限制

具体规则类型如下：

### 1.1 认领规则：rule\_type=claimRule

规则一：认领上限规则，rule\_code=claimUpper

针对各分层设置客户经理可认领的责任户上限

规则二：释放后认领规则，rule\_code=secondaryClaim

被系统定时任务自动释放的客户，同客户经理想再次认领时需要校验是否在时间范围内，仍在时间范围内的限制不可认领




### 1.2 自动释放规则：rule\_type=autoReleaseRule

 规则一：未跟进规则，rule\_code=notFollow

 X天内未跟进的释放

规则二：未转成商机客户，rule\_code=notSaleChane

在当前客户分层停留时间超过X天

以上，规则分类下的多个规则支持设置与或关系，即同时满足或任一满足

### 2.2 限制客户认领上限

当前仅限制大客业务的客户认领

    1. 公共池列表认领/大客机构一户式认领时，调用 /customer/claimCustomerManager[01-责任户认领接口](/read/book/70/174614)
        1. 接口内部限制了appCode=ptcrm 时走新的客户管理策略逻辑
        2. 取当前认领客户的客户分层
            1. 客户无分层 或 有分层且未配置分层的上限值的
                1. 不继续规则校验，可认领
            2. 客户有分层且有配置分层上限值
                1. 获取当前客户经理名下已是该分层的客户数量
                2. 已有的客户数量＜配置值，可认领
                3. 已有的客户数量≥配置值，不可认领
        3. 符合认领条件的客户继续划分客户逻辑，调用市场管理接口[16-批量认领&分配客户经理以及将客户划分到对应销售区域接口](/read/book/70/172780)

            1. appId 需要传对应的业务appId，如大客入口传人资大客应用的id
            2. operateType 传 1-认领
    2. 公共池列表分配客户，调用 /customer/allocateCustomerManager[05 责任户分配接口](/read/book/70/174610)
        1. 该接口仅人资分配使用，未限制appCode
        2. 获取分配客户的客户分层 - 同认领
        3. 获取分配客户经理名下已是该分层的客户数 - 同认领
        4. 比对满足规则的逻辑同认领

### 2.3 限制释放后认领天数

    1. 前提：和客户认领上限配置了且条件时，先校验客户上限
    2. 公共池列表认领/大客机构一户式认领时，调用 /customer/claimCustomerManager
        1. 接口内部限制了appCode=ptcrm 时走新的客户管理策略逻辑
        2. 取当前认领客户最后一次自动释放时间
            1. 调用市场管理接口[25-批量获取客户操作客户经理的划分日志](/read/book/70/172771)
                1. 需要限制客户经理，previousCustomerManagerId 传当前认领的客户经理id
                2. 只取自动释放的时间，限制操作类型，operateTypeList 传 4-自动释放
                3. 只取对应业务线产生的自动释放数据，区分大客和财代的自动释放，appId 需要传对应的业务appId，如大客入口传人资大客应用的id
            2. 客户无自动释放时间
                1. 不继续规则校验，可认领
            3. 客户有自动释放时间
                1.  将当前时间 和 自动释放时间进行日期转换，按日比较
                2. 当前日期-自动释放日期 ＞配置值，可认领
                3. 当前日期-自动释放日期 ≤ 配置值，不可认领
        3. 符合认领条件的客户继续划分客户逻辑，调用市场管理接口[16-批量认领&分配客户经理以及将客户划分到对应销售区域接口](/read/book/70/172780)

            1. appId 需要传对应的业务appId，如大客入口传人资大客应用的id
            2. operateType 传 1-认领
        4. 不符合释放后认领规则的，接口返回指定code，用于前端判断
            1. messageCode: "not.allow.claim.again"
    3. 公共池列表分配不受释放后认领规则限制
        1. 即客户经理认领不满足释放后认领规则时，可通过主管分配直接划分客户




```sql
-- 获取客户的最近自动释放时间
select *
-- max(create_date) 
from market_divide.md_region_customer_manager_log 
where customer_id = 'C28A4C204B894C0087F609456E80F71C' 
and app_id = '85915252930047202309081033247'
and operate_type in ('4')
order by id desc;
```










### 2.4 释放后二次认领工作流审批

    1. 场景：认领客户，调用 /customer/claimCustomerManager 时，判断接口返回的messageCode
        1. messageCode: "not.allow.claim.again"
        2. 入口：PC&mob公共池列表，PC&mob大客机构一户式
    2. 调用 /common/startProcessInstance 发起工作流申请
        1. itcrmweb\_work\_flow\_record 记录工作流申请，process\_code = 'claimCustomerManagerAgain'
        2. 限制同一客户经理同一客户只能有一个在申请中状态的工作流
            1. 查询itcrmweb\_work\_flow\_record表，解析工作流请求参数，同客户同客户经理已存在 flow\_status=0 的记录时，不可重复发起
    3. 工作流：释放后特殊认领 - claimCustomerManagerAgain
        1. 配置二级审批
        2. 一级审批驳回的，终止流程，审批不通过
        3. 一级审批通过的，流转到二级审批
            1. 二级审批驳回，终止流程，审批不通过
            2. 二级审批通过
    4. 监听工作流审批MQ：xbpmsweb-process-instance-end

itcrm\_mq\_record 表保存mq消费记录

        1. 审批不通过
            1. 不处理
        2. 审批通过
            1. 再次校验可认领规则
                1. 当前场景下不再校验释放后认领规则
                2. 仍需校验上限规则
            2. 符合认领规则
                1. 调用市场管理接口进行客户划分
                    1. 划分成功，给申请人推送客户划分成功的消息
                    2. 划分失败，给申请人推送客户划分失败的消息
            3. 不符合认领规则
                1. 给申请人推送客户划分失败的消息

### 2.6 自动释放定时任务

定时任务Id：CustomerManageStrategyReleaseTask

***规则里使用到的时间都精确到【日】比对，包括：认领时间、跟进时间、当前时间***

    1. 获取配置表里配置了自动释放规则的经营中心和大区
        1. 规则执行按经营中心+大区维度
        2. 如只配置了经营中心级的规则，大区配置成default的，需将经营中心转换成大区列表，按大区循环执行
            1. 从缓存获取经营中心下的大区
    2. 获取要计算的客户范围
        1. 按appCode+经营中心+大区获取可见销售区域范围（同公共池公共逻辑）
            1. 大客场景下是大区含下级的销售区域，即大区下的所有客户
        2. 批量查询销售区域范围内的客户分层
        3. 过滤客户所属分层未配置释放规则的客户
    3. 按客户判断是否符合自动释放规则
        1. 判断未跟进规则
        2. 判断未转商机客户规则
        3. 以上规则线上使用【或】条件，任一满足即释放
    4. 符合释放规则的客户调用市场管理释放客户，记录释放数据
        1. [17-批量释放客户经理](/read/book/70/172779)
            1. appId 需要传对应的业务appId，如大客自动释放传人资大客应用的id
            2. operateType 传 4-自动释放
        2. itcrm\_customer\_manage\_auto\_release\_data 表记录释放客户，当前客户经理和释放原因

#### 2.6.1 自动释放定时任务-未跟进规则

        1. 获取客户划分信息[01-根据客户id列表批量查询划分结果](/read/book/70/172794)
            1. 拿到客户的当前客户经理
            2. 客户无客户经理 或 无划分信息的直接结束，无需处理
        2. 调用市场管理接口 [25-批量获取客户操作客户经理的划分日志](/read/book/70/172771)获取客户在该客户经理名下最新的划分时间
            1. 不区分认领和分配（1-认领，2-分配），operateTypeList 传1,2
            2. 不限制其他业务入口产生的划分数据， appId 传NULL
        3. 判断认领时间和当前时间

            1. 当前时间-认领时间 ≤ 未跟进的配置
                1. 不释放
            2. 当前时间-认领时间 ＞ 未跟进的配置
                1. 获取该客户经理对客户跟进的最后一次跟进时间，限制客户经理+客户
                2. 未获取到跟进记录：释放
                3. 有获取到跟进记录
                    1. 当前时间-跟进时间 ≤ 未跟进的配置：不释放
                    2. 当前时间-跟进时间 ＞ 未跟进的配置：释放




```sql
-- 获取客户经理跟进责任户的最近跟进时间，因为大客只有中介，只关联中介表
select max(create_time)
from itcrm_follow_record_info where id in (
	select follow_record_id from itcrm_follow_record_institution_relation 
	where institution_id = '3367CD5A203A414D9E18F823C63891F8') -- 客户id
and follow_person  in ( '3e66b4c2f2d74535b43316cd72c61ccb'); -- 责任客户经理id

-- 获取客户的最近一次认领/分配时间
select max(create_date) 
from market_divide.md_region_customer_manager_log 
where customer_id = '587341F15EEF421684ABF867F5AA51F3' 
and customer_manager_id = '3e66b4c2f2d74535b43316cd72c61ccb'
and operate_type in ('1','2')
order by id desc;

-- 记录自动释放数据
select * from itcrm_customer_manage_auto_release_data;
```

#### 2.6.2 自动释放定时任务-未转商机客户规则

        1. 获取客户划分信息，客户的认领/分配时间 - 同未跟进规则
        2. 判断认领时间和当前时间
            1. 当前时间-认领时间 ≤ 未转商机客户的配置：不释放
            2. 当前时间-认领时间 > 未转商机客户的配置：释放
