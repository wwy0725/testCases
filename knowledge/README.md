# 业务知识库（小二当家）

> 本知识库为 testcase-generator subagent 写测试用例时提供**真实业务背景**，避免用例脱离实际。

## 📦 来源

- **原始来源**：17work 知识库节点 https://17work.dc.servyou-it.com/read/book/70/200181
- **拉取方式**：`npx @servyou-ai/17work-cli docs download 200181 --recursive`
- **同步脚本**：`scripts/sync-knowledge.ps1`（支持定时）
- **原始内容**：`knowledge/_raw/`（173 个文件，1.7MB，不动）
- **本次同步时间**：见 `knowledge/_sync.log`

## 🚀 使用方式（testcase-generator 必须遵守）

### 阶段 0：必须先读这三个文件

在生成任何测试用例**之前**，subagent 必须读：

1. **`knowledge/glossary.md`** — 确保用例用词与业务一致（不能用错术语）
2. **`knowledge/关键场景速查.md`** — 找到当前需求对应的"业务上下文"
3. **`knowledge/_raw/小二当家/小二当家_1 业务沉淀/小二当家_1.1 业务简介-业务架构.md`** — 业务全景（如果是新功能必读）

### 阶段 0.5：按需深入

- 写"机构一户式"相关用例 → 读 `knowledge/_raw/小二当家/小二当家_1 业务沉淀/小二当家_1.3 小二功能模块/机构一户式/`
- 写"跟进记录/转介绍"相关用例 → 读 `knowledge/_raw/小二当家/小二当家_1 业务沉淀/小二当家_1.3 小二功能模块/跟进记录/`
- 写"商机"相关用例 → 读 `knowledge/_raw/小二当家/小二当家_1 业务沉淀/小二当家_1.3 小二功能模块/商机/`
- 写"客户管理"相关用例 → 读 `知识库/.../客户管理策略（新）/`
- 写"权限/开关"相关用例 → 读 `知识库/.../小二当家权限及开关控制/`

### 写用例时引用业务术语

用例标题、前置条件、测试数据中出现的业务术语**必须与 glossary.md 一致**：

| 错误（脱离业务） | 正确（参考 glossary） |
|---|---|
| "客户 A" | "机构 XX"（如果是中介线）/ "企业 XX"（如果是自办税线） |
| "选择客户" | "选择跟进对象"（参见跟进记录.md 第 22 行） |
| "登录人" | "当前客户经理" |
| "客户列表" | "责任户"/"公共池"（取决于场景） |
| "新功能上线" | "管理端配置启用"（参见运营支撑管理-新跟进记录配置.md） |

## 🔄 同步策略

| 项 | 策略 |
|---|---|
| 同步方式 | 全量重新下载（覆盖 `_raw/`，约 1-2 分钟） |
| 推荐频率 | 每周一次（业务沉淀变化较慢） |
| 同步脚本 | `powershell scripts/sync-knowledge.ps1` |
| 自动化（可选） | Windows 任务计划程序 → 每周日 02:00 触发 |
| 增量检查 | 17work CLI 不支持增量，只能全量 |

## ⚠️ 注意事项

- `_raw/` 内容**不要手动编辑**——下次同步会被覆盖
- 如发现业务术语过时，先改 `glossary.md` 和 `关键场景速查.md`，再触发同步
- 如发现 `_raw/` 缺少重要文档，可能是 17work 上文档未发布给当前账号，找文档负责人补
- 同步前确保 `npx @servyou-ai/17work-cli` 已登录（OAuth 自动处理）
