# 业务术语表（小二当家）

> 写测试用例时，**所有业务术语必须与本表一致**。术语释义源自 17work 业务沉淀。

## 👤 角色

| 术语 | 定义 | 来源 |
|---|---|---|
| **客户经理** | 使用小二当家平台的公司员工（总部或分公司），是小二当家所有操作的执行人 | 业务架构.md L1 |
| **当前客户经理** | 当前登录人；也是"操作人" | 公共池.md |
| **责任户** | 划分给当前客户经理的客户，由其经营 | 业务架构.md L18 |
| **业务员** | 同"客户经理"（口语） | 跟进记录.md |

## 🏢 客户实体（四类）

| 术语 | 定义 | 业务线归属 |
|---|---|---|
| **企业（单位客户）** | 客户主类型之一，企业维度经营 | 自办税线 |
| **中介** | 代理机构 | 中介线 |
| **个代（个人代理）** | 个人代理 | 自办税线 / 中介线 |
| **用户（联系人）** | 联系人维度 | 自办税线 |
| **机构** | 中介 + 中介线的个代，统称 | 中介线 |

> 关键区分：自办税线的个代 → 个代一人式经营；中介线的个代 → 机构一户式经营（按代理机构维度）。

## 📱 核心页面（经营维度）

| 术语 | 定义 | 经营维度 |
|---|---|---|
| **一人式** | 展示用户详情的页面 | 用户维度 |
| **企业一户式** | 展示企业详情的页面 | 企业维度 |
| **机构一户式** | 展示中介线个代/中介详情的页面 | 机构维度 |
| **个代一人式** | 用户有个代身份时的页面，仍以用户维度经营 | 用户维度 |
| **多企业一户式** | 经营体（多企业）维度 | 经营体维度 |

## 🔗 关键关系

| 术语 | 定义 |
|---|---|
| **人企关系** | 企业和联系人之间的关系，通过联系人可联系到这家企业 |
| **委托关系** | 企业和代理之间的关系，企业委托代理办业务（被个代或中介代理） |
| **主从关系** | 企业间的母子/主从关系（机构一户式有专门页签） |
| **联合经营** | 跨客户经理的合作模式 |

## 📊 业务模块

| 术语 | 定义 | 关键文档 |
|---|---|---|
| **跟进记录** | 客户经理跟客户进行营销/续费等经营动作后记录的过程 | `跟进记录/跟进记录.md` |
| **转介绍** | 客户介绍新客户的动作，跟进记录的关联组件之一（PC+移动） | `机构一户式/转介绍列表.md`、`跟进记录/跟进记录.md` L49 |
| **销售机会（商机）** | 从接触客户到赢单的全过程记录 | `商机/商机.md` |
| **签到/签退** | 业务员外勤定位签到 | `签到签退/` |
| **公共池** | 客户经理可触达的"未分配"客户集合，按层级（总部/经营中心/大区/网点）划分 | `公共池/公共池.md` |
| **责任户管理** | 当前客户经理经营范围内的客户管理 | `责任户管理/责任户管理.md` |
| **客户管理策略** | 不同类型客户的差异化经营策略（财代/大客/普通） | `客户管理策略（新）/` |
| **续费管理** | 客户续费流程 | `续费管理.md` |
| **公共查询** | 全局客户查询能力 | `公共查询.md` |
| **客户定位** | 通过各种维度（手机号/名称等）找客户 | `客户定位.md` |
| **首页搜索** | 全局搜索入口 | `首页搜索含移动端查企业.md` |

## 🎯 经营动作

| 术语 | 定义 |
|---|---|
| **跟进对象** | 跟进的目标实体（中介/个代/联系人/单位用户/多企业，多选或单选） |
| **跟进方式** | 电话/上门/企微/其他 |
| **跟进结果** | 跟进完成后的状态标签，决定后续组件排序 |
| **经营动作** | 跟进的业务行为（推荐产品/介绍活动/催促续费等） |
| **下次跟进时间** | 预约下一次的跟进时间 |
| **联系人/沟通记录** | 与客户的联系历史 |
| **看问讲** | 一人式的核心经营动作序列（看客户信息→问需求→讲方案） |

## 🔧 权限与配置

| 术语 | 定义 |
|---|---|
| **管理端配置** | 运营支撑管理端控制功能的开关/字段必填/排序等 |
| **业务权限** | 当前客户经理可执行的操作（authority-list 接口） |
| **商务开关** | 商务侧控制的功能开关（例：机构一户式"预收款余额"模块） |
| **大区** | 销售区域划分层级（总部/经营中心/大区/网点 4 级） |
| **场景 code** | 业务方调用跟进组件时传入的业务标识（销售机会/待办等） |
